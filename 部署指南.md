# IoT智能监控系统 - 本地部署运行指南

## 一、环境要求

| 软件 | 版本要求 | 说明 |
|------|----------|------|
| JDK | 11 或 17 | 推荐 JDK 11 |
| Maven | 3.6+ | 构建后端项目 |
| MySQL | 8.0+ | 数据库服务 |
| Node.js | 16+ | 运行前端（已安装v22） |
| npm | 8+ | 前端包管理器（已安装v10） |

## 二、环境安装步骤

### 2.1 安装 JDK 11
1. 下载：https://www.oracle.com/java/technologies/javase/jdk11-archive-downloads.html
   或使用 Adoptium：https://adoptium.net/temurin/releases/?version=11
2. 安装后配置环境变量：
   ```
   JAVA_HOME = C:\Program Files\Java\jdk-11
   Path 中添加 %JAVA_HOME%\bin
   ```
3. 验证：打开命令行输入 `java -version`

### 2.2 安装 Maven
1. 下载：https://maven.apache.org/download.cgi （Binary zip）
2. 解压到 `C:\Program Files\apache-maven-3.9.6`
3. 配置环境变量：
   ```
   M2_HOME = C:\Program Files\apache-maven-3.9.6
   Path 中添加 %M2_HOME%\bin
   ```
4. 验证：`mvn -version`

### 2.3 安装 MySQL 8.0
1. 下载：https://dev.mysql.com/downloads/mysql/
2. 安装时设置 root 密码为 `root`（与 application.yml 一致）
3. 或者安装后修改 `iot-backend/src/main/resources/application.yml` 中的密码

## 三、数据库初始化

### 方式一：命令行导入
```bash
mysql -u root -p < D:\python-test\IoTAgentGuard\iot-backend\sql\init.sql
```

### 方式二：手动导入
1. 打开 MySQL 命令行或 MySQL Workbench
2. 执行 `iot-backend/sql/init.sql` 中的 SQL 语句

## 四、启动后端

```bash
cd D:\python-test\IoTAgentGuard\iot-backend
mvn clean package -DskipTests
java -jar target/iot-guard-1.0.0.jar
```

后端启动成功后访问：http://localhost:8080

## 五、启动前端

```bash
cd D:\python-test\IoTAgentGuard\iot-frontend
npm install
npm run dev
```

前端启动成功后访问：http://localhost:5173

## 六、系统使用

1. 打开浏览器访问 http://localhost:5173
2. 使用默认账号登录：
   - 用户名：`admin`
   - 密码：`admin123`
3. 登录后进入仪表盘，可查看设备状态、数据统计

## 七、常见问题

### Q: MySQL连接失败
A: 检查 MySQL 服务是否启动，application.yml 中的数据库配置是否正确

### Q: 后端启动端口被占用
A: 修改 application.yml 中的 `server.port`

### Q: 前端 npm install 报错
A: 尝试删除 node_modules 和 package-lock.json，重新执行 `npm install`

### Q: 跨域报错
A: 后端已配置全局跨域（CorsConfig），前端 vite.config.js 已配置代理

### Q: 登录提示用户不存在
A: 确认已执行 init.sql 导入初始数据，注意 admin 的密码是 BCrypt 加密后的值
